[FastQC](https://www.bioinformatics.babraham.ac.uk/projects/fastqc/) and [MultiQC](https://seqera.io/multiqc/) are the industry-standard tools for assessing and visualising the quality of raw high-throughput sequencing (NGS) data (e.g., FASTQ, BAM, or SAM files). FastQC generates detailed QC reports for individual files, while MultiQC aggregates these metrics into a single, interactive, and comprehensive dashboard.

</div>

<div class="justify">

Here is an overview of how these tools work together and what to look for in your analysis:

</div>

## 1. The Core Tool: FastQC

FastQC evaluates raw sequence data across ten distinct modules, providing a quick assessment **(Pass/Warning/Fail)** for metrics such as:

- *Per Base Sequence Quality*: Checks if the sequencing quality scores (Phred scores) remain high across the entire length of the read.
- *Sequence Duplication Levels*: Helps identify potential PCR over-amplification or low-complexity libraries.
- *Adapter Content*: Alerts you if raw sequencing adapters have not been fully trimmed from the ends of the reads.

### Basic Usage
To run FastQC on a single or multiple FASTQ files, use the following terminal command:

```bash
fastqc read1_R1.fq.qz read1_R2.fq.qz
```

:::{tip}
You can use `fastqc *.fq.gz` to process every file in your directory automatically or you can use loops to iterate into each file and process all fastq files with the extension `.fq.gz`.
:::

```bash
for i in sample /path/to/fastq/reads/ ; do fastqc '$i' -o /path/to/fastqc/output;
done
```

## 2. The Aggregator: MultiQC

<div class="justify">

Reviewing dozens or hundreds of individual FastQC HTML reports is tedious. MultiQC solves this by scanning your results directory, extracting the metadata and statistics, and compiling them into a single streamlined HTML file that is easy to share and interpret.

</div>

### Basic Usage
To use MultiQC, simply navigate to the folder containing your FastQC output files and run:

```bash
multiqc . -o path/to/multiqc/output
```

## Bonus

<div class="justify">

Key Metrics to Monitor When reviewing your MultiQC report, pay close attention to these common warning signs:

</div>
- *Sequence Length Distribution*: If reads vary heavily in length, you may need to trim low-quality sequences or adapter fragments using a tool like [Trim Galore](https://www.bioinformatics.babraham.ac.uk/projects/trim_galore/) or [fastp](https://github.com/opengene/fastp).

- *Per Sequence Quality Scores*: A significant drop in quality toward the 3' end of the reads is common in Illumina sequencing. If the average score drops below 20 (indicating a 1% probability of an incorrect base call), trimming is necessary.

- *Overrepresented Sequences*: This usually indicates either adapter dimers or highly abundant transcripts (e.g., ribosomal RNA in RNA-Seq).

Additional parameters should be taken into account depending on the type of genomic data you are dealing with.
